# 폼 교체 시스템

**기능 번호**: 017
**브랜치**: `017-form-swap-system`
**작성일**: 2025-11-29
**상태**: 명세 완료

---

## 1. 개요

### 1.1 기능 설명
플레이어가 최대 2개의 마법 폼을 보유하고 Q키로 즉시 교체할 수 있는 시스템입니다. 각 폼은 고유한 외형, 스탯, 스킬을 가지며, 전투 상황에 따라 전략적으로 교체하여 플레이합니다.

### 1.2 사용자 가치
- **전략적 깊이**: 상황에 맞는 폼 선택으로 전투 전략 수립
- **수집 욕구**: 다양한 폼 획득 및 조합 탐색
- **플레이 스타일 다양화**: 폼마다 다른 플레이 경험
- **성장 만족감**: 폼 각성을 통한 강화 체감

### 1.3 범위
- **포함**: 폼 데이터, 폼 교체, 폼 획득, 폼 각성, UI
- **제외**: 폼 합성/조합, 폼 커스터마이징

---

## 2. 사용자 시나리오

### 2.1 주요 시나리오: 폼 교체

**전제조건**: 플레이어가 2개의 폼을 보유

**흐름**:
1. 플레이어가 전투 중 Q키 입력
2. 현재 폼이 비활성화되고 다른 폼이 활성화
3. 플레이어 외형이 새 폼으로 즉시 변경
4. 스탯이 새 폼 기준으로 변경
5. 스킬 슬롯이 새 폼의 스킬로 교체
6. 화면에 폼 교체 이펙트 표시
7. 교체 쿨다운 시작 (기본 5초)

### 2.2 시나리오: 새 폼 획득

**전제조건**: 플레이어가 던전에서 폼 아이템 발견

**흐름**:
1. 드롭된 폼 아이템과 상호작용
2. 폼 정보 팝업 표시 (이름, 등급, 스탯, 스킬 설명)
3. **보유 슬롯이 비어있는 경우**: 즉시 획득
4. **보유 슬롯이 가득 찬 경우**: 교체할 폼 선택 UI 표시
5. 플레이어가 버릴 폼 선택 또는 획득 취소
6. 선택한 폼은 바닥에 드롭

### 2.3 시나리오: 폼 각성

**전제조건**: 동일한 폼을 2개 이상 획득

**흐름**:
1. 같은 종류의 폼 획득 시 자동 각성 진행
2. 폼 등급 상승 (일반 → 레어 → 유니크 → 레전더리)
3. 스탯 증가 및 스킬 강화
4. 각성 이펙트 및 축하 메시지 표시
5. 미니맵/HUD에 각성 완료 표시

### 2.4 시나리오: 폼 정보 확인

**전제조건**: 플레이어가 인벤토리 열기 (I키)

**흐름**:
1. 보유 중인 폼 2개 표시
2. 폼 선택 시 상세 정보 표시
   - 이름, 등급, 현재 각성 단계
   - 기본 스탯 (공격력, 공격속도, 이동속도 등)
   - 스킬 1, 스킬 2 설명 및 쿨다운
   - 각성 효과 미리보기

## 6. 핵심 엔티티

### 6.1 FormData (ScriptableObject)
- formId: 고유 ID
- formName: 폼 이름
- description: 설명
- formType: 타입 (Basic, Flame, Frost, Thunder, Dark)
- baseRarity: 기본 등급
- baseStats: 기본 스탯 (공격력, 공격속도, 이동속도, 점프력, 체력보너스)
- skill1: 첫 번째 스킬 (SkillData 참조)
- skill2: 두 번째 스킬 (SkillData 참조)
- sprite: 아이콘 스프라이트
- animatorController: 애니메이션 컨트롤러
- swapEffect: 교체 시 이펙트 프리팹

### 6.2 FormInstance (런타임 데이터)
- formData: FormData 참조
- currentRarity: 현재 등급 (각성 상태)
- awakeningLevel: 각성 단계 (0~3)
- currentStats: 현재 스탯 (각성 보너스 적용)

### 6.3 FormManager
- currentForm: 현재 활성 폼
- reserveForm: 대기 폼
- swapCooldown: 교체 쿨다운 상태
- canSwap: 교체 가능 여부

## 7. 가정 사항

1. 기존 PlayerStats 시스템을 활용하여 스탯 적용
2. 기존 SkillSystem을 활용하여 스킬 교체

4. 애니메이션 컨트롤러는 폼별로 별도 제작
