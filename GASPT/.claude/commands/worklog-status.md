# 작업 로그 현황 (읽기 전용)

출근 시 이전 작업 상태를 빠르게 파악하기 위한 읽기 전용 명령입니다.
**파일 수정 없이** 현재 상태만 요약하여 보여줍니다.

---

## 📋 수행 작업

### 1. Git 상태 확인 (수정 없음)
다음 명령을 실행하여 정보 수집:
- `git log --oneline -10` - 최근 10개 커밋
- `git branch --show-current` - 현재 브랜치
- `git status --short` - 스테이지되지 않은 변경사항

### 2. 작업 로그 읽기 (수정 없음)
다음 파일들을 **읽기만** 수행:
- `docs/work-logs/programming/LATEST.md`
- `docs/work-logs/design/LATEST.md` (존재 시)
- `docs/work-logs/README.md`

### 3. 진행 중인 Spec 확인 (수정 없음)
- `specs/` 폴더에서 진행 중인 spec 확인
- `tasks.md` 파일에서 미완료 태스크 확인

---

## ✅ 출력 형식

아래 형식으로 **요약 보고서** 출력:

```
📊 작업 현황 요약 (YYYY-MM-DD)
================================

🌿 브랜치: master

📝 최근 커밋 (최근 5개):
1. [해시] 커밋 메시지 1
2. [해시] 커밋 메시지 2
3. [해시] 커밋 메시지 3
...

🔄 현재 진행 중:
- [Spec 이름] - 진행률 XX%
- 세부 작업: ...

✅ 최근 완료된 작업:
- 작업명 1 (날짜)
- 작업명 2 (날짜)

📌 다음 작업 후보:
- [ ] 후보 1
- [ ] 후보 2
- [ ] 후보 3

⚠️ 스테이지되지 않은 변경사항:
- 파일1
- 파일2

💡 오늘 추천 작업:
가장 우선순위가 높은 작업을 간단히 제안
```

---

## 🎯 핵심 원칙

1. **읽기 전용**: 어떤 파일도 수정하지 않음
2. **빠른 요약**: 핵심 정보만 간결하게 표시
3. **컨텍스트 제공**: 어제 마지막 작업이 무엇이었는지 파악 가능
4. **다음 단계 제안**: 오늘 시작할 작업 추천

---

## 📖 사용 시나리오

### 출근 시
```bash
/worklog-status
```
→ 어제 어디까지 했는지, 오늘 뭘 해야 하는지 빠르게 파악

### 점심 후 복귀 시
```bash
/worklog-status
```
→ 오전에 하던 작업 컨텍스트 복구

### 작업 전환 시
```bash
/worklog-status
```
→ 현재 프로젝트 상태 확인 후 다른 브랜치/작업으로 전환

---

## ⚠️ 주의사항

- 이 명령은 **정보 조회만** 수행합니다
- 작업 로그를 **업데이트하려면** `/update-worklog` 사용
- Git 상태 변경 없음 (commit, push 등 수행 안 함)

---
